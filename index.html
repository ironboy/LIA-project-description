LIA-matcharen: Ett projekt om LIA, genomfört av LIA-studerande

22 april, år 2025 (vecka 17), Thomas Frank

Den 13 januari påbörjade jag, på uppdrag av KYH Anywhere, projektledningen av ett projekt med avsikt att bygga en webbaserad applikation som matchar LIA-platser mot studerande som söker LIA.

Varje vecka lägger jag, på uppdrag av KYH Anywhere, i snitt 4-5 timmar på ledning av projektet. Detta sker huvudsakligen i form av projektmöten, men också en i form av teknisk hjälp, samt val och integration av teknik och verktyg.

I projektet deltar, på heltid, 14 LIA-studerande från tre olika KYH-utbildningar.

De studerande kommer från följande KYH Anywhere-utbildningar (samtliga med start år 2023):

  • Mobilapplikationsutvecklare (2 studerande, vecka 3-20)
  • Frontend-utvecklare (7 studerande, vecka 5-20)
  • PVT23, Programvarutestare (5 studerande, vecka 9-23)

Vårt projektmål är att leverera en fungerande prototyp.

Projektbeskrivning från KYH Anywhere

Theresa Rosbäck på KYH Anywhere har lämnat följande projektbeskrivning som beställningsunderlag:

Utveckling av Interaktivt Matchningssystem för LIA-placeringar

Bakgrund: Vår organisation eftersträvar en effektiviserad process för att matcha studerande med företag för Lärande i Arbete (LIA). Denna tjänst syftar till att förenkla och förbättra matchningsprocessen genom ett interaktivt och användarvänligt system.
Projektbeskrivning: Vi önskar utveckla en webbaserad plattform som möjliggör för företag och studerande att enkelt hitta samarbetsmöjligheter för LIA. Plattformen ska omfatta tre huvudsakliga användartyper: företag, studerande, och administratörer från vår organisation (KYH). Projektet ska genomföras med hjälp av våra studerande från utbildningarna applikationsutvecklare, frontendutvecklare samt programvarutestare.
Företagskonto:

  • Gränssnitt för företag att skapa profiler med företagsinformation, kompetensbehov och tillgänglighetsperioder för LIA. -
  • Möjlighet för automatisk matchning med studerande, baserat på definierade kriterier.
  • Alternativ för vår personal att manuellt lägga till företagsprofiler vid behov.
    Studerandekonto:
  • Gränssnitt för studerande att upprätta profiler inkluderande CV och utbildningsinformation.
  • Funktion för att få förslag på matchande företag och möjlighet till direktkontakt.
    Adminkonto:
  • Administrativa verktyg för att övervaka och hantera företags- och studerandekonton.
  • Åtkomst till statistik över användare, matchningar och systemaktivitet.
  • Verktyg för att moderera innehåll och hantera konton.
    Syfte och Mål: Målet med tjänsten är att effektivisera matchningsprocessen mellan studerande och företag för LIA. Detta skall åstadkommas genom ett intuitivt gränssnitt, avancerade matchningsalgoritmer och administrativa verktyg för ökad översikt och kontroll.
    Leverans och Tidsplan: Projektet börjar med en analys- och designfas, följt av utvecklingen av systemet. Slutligen genomförs tester och implementering. En preliminär tidsram för projektet är vecka 3 – vecka 20 2025 Theresa Rosbäck

Möjlighet att uppnå projektmålen?

  • Då Node Hill AB inte kan gå in och genomföra själva utvecklingsarbetet, utan vi/jag är anlitad i en projektledande roll med begränsat anta timmar (4-5) att lägga per vecka, så kan vi/jag inte garantera slutresultatet. Detta beror i hög grad på de studerandes kimpetens och deras arbetsinsats under LIA:n.
  • Vad jag kan göra är att leda projektet professionellt och lyhört - vilket innefattar att lyssna på deltagarnas åsikter, samt ta avgörande beslut kring teknikval och prioritering av arbetsuppgifter.
  • Från var vi befinner oss i vecka 17 ser det ut som om vi kommer att kunna visa upp en fungerande prototyp/tidig alfaversion. Då denna är skapad med moderna webbteknologier är den möjlig att låta profesionella utvecklare (t.ex. anställda från Node Hill AB) bygga vidare på för att uppnå en färdig proukt.

Val av teknologier och verktyg

Tidigt i projekt presenterade jag färdiga teknikval för de studerande.

Teknik- och verktygsval från start

Git för versionshantering.

  • GitHub som molnserver för vårt Git-repo.
  • GitHub Projects för SCRUM-board.
  • GitHub Wiki för att dela information om olika aspekter av projektet.
  • Slack för kommunikation och samarbete mellan projektdeltagarna (chat, videomöten etc).
  • Google Meet för gemensamma veckomöten ledda av mig.
  • Visual Studio Code som rekommenderad IDE (kodeditor).
  • Figma för att skissa det grafiska gränssnittet, med utgångspunkt från ett mobile first-tänk.
  • JavaScript som programspråk, för både frontend och backend.
  • Node.js och Express som ramverk för backend.
  • MySQL som databasserver.
  • React med Vite.js som utvecklings- och byggverktyg för frontendutveckling.
  • React Router för frontendrouting.
  • React Easier för att underlätta state-hantering i React.
  • Bootstrap som CSS-ramverk för responsiv design.
  • React Bootstrap för komponentbaserad integration av React och Bootstrap.
  • SASS som CSS-preprocessor för att skriva egen CSS-kod och göra ändringar av de variabler som styr Bootstraps utseende.

Ytterligare val under projektets gång

Utöver de inledande valen:

  • Playwright som huvudsakligt automatiserat testverktyg. Valet gjordes när testarna började arbeta i projektet
  • D3.js för visualiseringar. Valet gjordes när en av utvecklarna presenterade en idé om visualiseringar med hjälp av detta verktyg (diagram kring användnning av systemet, kartor med LIA-platser markerade etc).
